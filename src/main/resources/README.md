## flywayDB 数据库自动迁移
### 文件命名规范
0. 迁移文件格式 `V${version}__${file name}.sql`, 其中，`version`必须比上一个版本大。
1. 仅需要被执行一次的SQL命名以大写的`V`开头，后面跟上 `0~9` 数字的组合数字之间可以用点 `.` 或者下划线 `_` 分割开，然后再以两个下划线分割，其后跟文件名称，最后以`.sql` 结尾。比如，`V0.1.0__database.sql`、 `V0.1.1__database.sq;`。
2. 可重复运行的SQL,则以大写的 `R` 开头，后面再以两个下划线分割，其后跟文件名称，最后以`.sql` 结尾。 比如: `R_truncate_user_dml.sql` .
3. 其中，`V` 开头的SQL执行优先级要比`R` 开头的SQL优先级高。
4. 仅允许有一个 `V 开头的迁移文件，
### 版本号比较

> 版本号的比较规则如下：

- 版本号的格式为 `x.y.z`，其中 `x`、`y` 和 `z` 均为非负整数 `大版本.小版本.修订版本`。
- 连个版本号比较：
  - 如果 `x` 不同，则 `x` 较大，如果 `x` 相等，则比较 `y`，如果 `y` 不同，则 `y` 较大，如果 `y` 相等，则比较 `z`。
